# Define XML 项目管理系统 - 开发需求文档

## 1. 项目概述
1、用户登录进去后，创建项目，把对应数据库的数据与界面项目列表绑定，同时也把对应的项目文件夹创建好直接复制demo即可； 2、进入项目看到项目配置信息， 3、可多选选择标准（SDTM/ADAM/SEND) 选择几个给出几个上传文档的界面,例如SDTM（aCRF，P21空Spec，项目Spec，xpt）， 把上传的文件管理项目文件夹下的Upload存档， 同时也把对应的define中的当前使用的文件更新(如aCRF文件夹下aCRF.PDF，P21空Spec文件夹下P21空Spec.xlsx，项目Spec文件夹下项目Spec.xlsx，xpt文件夹下xpt.xlsx)； 把P21空Spec更新到对应数据库，项目Spec也更新到数据库； 点击处理aCRF，调用python程序把aCRF里的注释获取； 点击生成vlm调用python下对应程序； 点击生成code_list调用python下对应程序； 把生成好的VLM和CodeList更新到对应的数据库中，这样界面上就可以对其进行交互操作调序，修改文字等； 最后把对应的VLM和CodeList整合到项目Spec下对应的sheet ；点击更新page，调用python程序可以把整合好的项目Spec更新里面的page；这样把需要准备的文件都准备好了。 在Define制作界面，选择制作SDTM，把需要的一些参数填好后，最后点击生成就会调用python，生成最终的define.xlsx，并下载到本地。
### 1.1 项目背景
Define XML 项目管理系统是一个专门用于临床试验数据管理的平台，支持SDTM、SEND、ADAM等国际标准，提供项目创建、文件上传、数据处理、Define.xml生成等功能。

### 1.2 项目目标
- 提供简洁易用的项目管理界面
- 支持多种临床数据标准（SDTM/SEND/ADAM）
- 实现文件上传、处理、转换功能
- 自动生成符合FDA要求的Define.xml文件
- 确保数据安全和用户权限控制

## 2. 功能需求

### 2.1 项目管理模块

#### 2.1.1 项目创建
- **功能描述**: 用户可以创建新的临床试验项目
- **输入字段**:
  - 项目ID（必填，唯一标识）
  - 项目名称（必填）
  - 方案编号（可选）
  - 方案名称（可选）
  - 方案版本（可选）
  - 方案版本日期（可选）
  - 申办方/赞助方（可选）
- **业务规则**:
  - 项目ID必须唯一
  - 项目ID格式建议：字母、数字、下划线组合
  - 创建时自动关联当前用户
  - 自动创建项目文件夹结构

#### 2.1.2 项目列表
- **功能描述**: 显示用户有权限的所有项目
- **显示信息**:
  - 项目ID
  - 项目名称
  - 方案编号
  - 方案名称
  - 方案版本
  - 创建时间
  - 状态（活跃/归档）
- **操作功能**:
  - 进入项目
  - 删除项目
  - 归档/激活项目

#### 2.1.3 项目权限
- **权限控制**: 用户只能看到和操作自己创建的项目
- **系统项目**: 系统预置DEFAULT和P000_Demo模板项目

### 2.2 项目配置模块

#### 2.2.1 标准配置
- **编码格式**: UTF-8, GBK
- **使用语言**: 中文(CN), 英文(EN)
- **标准类型**: SDTM, SEND, ADAM
- **标准版本**:
  - SDTM: SDTM-IG3.2, SDTM-IG3.3, SDTM-IG3.4
  - SEND: SEND3.0, SEND3.1, SEND3.1.1
  - ADAM: ADAM-IG1.0, ADAM-IG1.1, ADAM-IG1.2, ADAM-IG1.3
- **CT版本**: CT2023-12-17, CT2024-03-27, CT2024-12-20
- **标准语言**: 中文标准, 英文标准（可多选）

### 2.3 文件上传模块

#### 2.3.1 支持的文件类型
1. **aCRF文件**（仅SDTM需要）
   - 格式：PDF
   - 用途：注释CRF，用于变量映射

2. **P21空Spec文件**
   - 格式：Excel (.xlsx, .xls)
   - 用途：P21标准的空白规格说明

3. **项目Spec文件**
   - 格式：Excel (.xlsx, .xls)
   - 用途：项目特定的数据规格说明

4. **XPT文件**
   - 格式：SAS传输文件 (.xpt)
   - 用途：临床数据文件

#### 2.3.2 文件上传要求
- 支持拖拽上传
- 文件大小限制：每个文件不超过100MB
- 同时上传文件数量限制：每种类型限制1个文件
- 上传失败自动重试机制
- 上传进度显示

#### 2.3.3 文件处理状态
- **上传状态**: uploading（上传中）, uploaded（已上传）, failed（上传失败）
- **处理状态**: pending（待处理）, processing（处理中）, completed（完成）, failed（失败）

### 2.4 数据管理模块

#### 2.4.1 Spec数据管理
- **数据域管理**: 支持DM, AE, CM等标准域
- **变量定义**: 变量名、标签、类型、长度等
- **CDISC规范**: 符合CDISC标准的变量定义
- **代码列表**: 受控术语和代码列表管理

#### 2.4.2 VLM管理
- **变量级元数据**: 支持复杂的WHERE条件
- **标签管理**: 多语言标签支持
- **分类管理**: 变量分类和排序

#### 2.4.3 CodeList管理
- **代码定义**: 代码值和描述
- **版本控制**: 代码列表版本管理
- **多语言支持**: 中英文代码描述

### 2.5 Define.xlsx生成模块

#### 2.5.1 Define制作界面
- **功能描述**: 提供可视化界面生成符合FDA要求的Define.xml文件
- **标准选择**:
  - 支持SDTM标准类型选择
  - 自动加载对应标准的参数配置模板
- **参数配置区域**:
  - 项目基本信息（项目ID、项目名称、方案编号等）
  - 标准版本信息（SDTM版本、CT版本）
  - 文件路径配置（输入文件路径、输出路径）
  - 自定义参数设置

#### 2.5.2 参数配置要求
- **必填参数**:
  - 项目ID
  - 标准类型（SDTM/SEND/ADAM）
  - 标准版本
  - 输入文件路径
  - 输出路径
- **可选参数**:
  - 方案编号
  - 申办方信息
  - 自定义标题
  - 语言设置（中文/英文）
- **参数验证**:
  - 实时参数有效性检查
  - 文件路径存在性验证
  - 参数格式规范验证

#### 2.5.3 生成功能
- **生成流程**:
  1. 参数配置完成后点击"生成Define.xlsx"按钮
  2. 系统调用后端API接口
  3. 后端调用Python脚本进行Define.xlsx生成
  4. 实时显示生成进度和状态
  5. 生成完成后提供下载链接
- **Python集成**:
  - 通过Java Process调用Python脚本
  - 支持参数传递和结果返回
  - 异常处理和错误信息反馈
- **文件输出**:
  - 生成标准的define.xlsx文件
  - 符合FDA Define-XML 2.0/2.1规范
  - 支持自定义输出路径
  - 提供文件下载功能

#### 2.5.4 进度监控
- **实时状态**: 显示当前生成进度百分比
- **状态提示**:
  - 准备中（参数验证）
  - 生成中（Python脚本执行）
  - 完成（文件生成成功）
  - 失败（错误信息显示）
- **日志记录**: 详细的生成过程日志
- **错误处理**: 友好的错误提示和解决建议
