# Define XML 数据库设计文档

## 1. 数据库概述

### 1.1 数据库信息
- **数据库名称**: define_db
- **数据库版本**: MySQL 8.0+
- **字符集**: utf8mb4
- **排序规则**: utf8mb4_unicode_ci
- **存储引擎**: InnoDB

### 1.2 设计原则
- **数据完整性**: 通过外键约束保证数据一致性
- **性能优化**: 合理设计索引，优化查询性能
- **可扩展性**: 预留扩展字段，支持业务增长
- **安全性**: 用户权限隔离，数据访问控制
- **标准化**: 遵循数据库设计规范

## 2. 核心表结构设计

### 2.1 项目管理表 (project)

#### 2.1.1 表结构
```sql
CREATE TABLE `project` (
    `id` BIGINT(20) NOT NULL AUTO_INCREMENT COMMENT '主键ID',
    `project_id` VARCHAR(50) NOT NULL COMMENT '项目ID',
    `project_name` VARCHAR(200) COMMENT '项目名称',
    `protocol_number` VARCHAR(100) COMMENT '方案编号',
    `protocol_name` VARCHAR(200) COMMENT '方案名称',
    `protocol_version` VARCHAR(20) COMMENT '方案版本',
    `protocol_date` DATE COMMENT '方案版本日期',
    `sponsor` VARCHAR(200) COMMENT '申办方/赞助方',
    `username` VARCHAR(100) COMMENT '用户名',
    `status` TINYINT(1) DEFAULT 1 COMMENT '状态：1-活跃，0-归档',
    `created_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `updated_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    `deleted` TINYINT(1) DEFAULT 0 COMMENT '删除标记：0-未删除，1-已删除',
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_project_id` (`project_id`),
    KEY `idx_protocol_number` (`protocol_number`),
    KEY `idx_status` (`status`),
    KEY `idx_deleted` (`deleted`),
    KEY `idx_created_time` (`created_time`),
    KEY `idx_username` (`username`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='项目管理表';
```

#### 2.1.2 字段说明
| 字段名 | 类型 | 长度 | 允许空 | 默认值 | 说明 |
|--------|------|------|--------|--------|------|
| id | BIGINT | 20 | NO | - | 主键ID，自增 |
| project_id | VARCHAR | 50 | NO | - | 项目ID，业务主键，全局唯一 |
| project_name | VARCHAR | 200 | YES | NULL | 项目名称 |
| protocol_number | VARCHAR | 100 | YES | NULL | 方案编号 |
| protocol_name | VARCHAR | 200 | YES | NULL | 方案名称 |
| protocol_version | VARCHAR | 20 | YES | NULL | 方案版本 |
| protocol_date | DATE | - | YES | NULL | 方案版本日期 |
| sponsor | VARCHAR | 200 | YES | NULL | 申办方/赞助方 |
| username | VARCHAR | 100 | YES | NULL | 项目所有者用户名 |
| status | TINYINT | 1 | NO | 1 | 项目状态：1-活跃，0-归档 |
| created_time | DATETIME | - | NO | CURRENT_TIMESTAMP | 创建时间 |
| updated_time | DATETIME | - | NO | CURRENT_TIMESTAMP | 更新时间，自动更新 |
| deleted | TINYINT | 1 | NO | 0 | 逻辑删除标记：0-未删除，1-已删除 |

#### 2.1.3 索引设计
- **主键索引**: `PRIMARY KEY (id)` - 数据库主键
- **唯一索引**: `uk_project_id (project_id)` - 保证项目ID全局唯一
- **普通索引**:
  - `idx_protocol_number (protocol_number)` - 按方案编号查询
  - `idx_status (status)` - 按状态筛选
  - `idx_deleted (deleted)` - 逻辑删除过滤
  - `idx_created_time (created_time)` - 按创建时间排序
  - `idx_username (username)` - 按用户查询项目


#### 3.1.2 核心字段说明
- **domain**: 数据域名称，如DM(人口学)、AE(不良事件)、CM(合并用药)等
- **variable**: 变量名称，如USUBJID、AETERM、CMTRT等
- **core**: 核心级别
  - Req(Required): 必需的
  - Exp(Expected): 期望的
  - Perm(Permissible): 允许的
- **origin**: 数据来源
  - CRF: 来自病例报告表
  - Assigned: 分配的值
  - Derived: 派生得出

## 5. 视图设计

### 5.1 项目域统计视图 (v_project_spec_domains)

```sql
CREATE OR REPLACE VIEW `v_project_spec_domains` AS
SELECT
  `project_id`,
  `domain`,
  COUNT(*) as `variable_count`,
  COUNT(CASE WHEN `core` = 'Req' THEN 1 END) as `required_count`,
  COUNT(CASE WHEN `core` = 'Exp' THEN 1 END) as `expected_count`,
  COUNT(CASE WHEN `core` = 'Perm' THEN 1 END) as `permissible_count`,
  MIN(`created_time`) as `first_created`,
  MAX(`updated_time`) as `last_updated`
FROM `sas_project_spec`
GROUP BY `project_id`, `domain`
ORDER BY `project_id`, `domain`;
```

### 5.2 变量使用频率统计视图 (v_variable_usage_stats)

```sql
CREATE OR REPLACE VIEW `v_variable_usage_stats` AS
SELECT
  `variable`,
  `label`,
  `type`,
  COUNT(DISTINCT `project_id`) as `project_count`,
  COUNT(DISTINCT `domain`) as `domain_count`,
  GROUP_CONCAT(DISTINCT `domain` ORDER BY `domain`) as `used_in_domains`,
  COUNT(*) as `total_usage`
FROM `sas_project_spec`
GROUP BY `variable`, `label`, `type`
HAVING COUNT(*) > 1
ORDER BY `total_usage` DESC, `variable`;
```

## 6. 索引优化策略

### 6.1 查询优化索引

#### 6.1.1 常用查询模式
1. **按用户查询项目**: `username` + `status` + `deleted`
2. **按项目查询配置**: `project_id`
3. **按项目查询文件**: `project_id` + `file_type`
4. **按域查询变量**: `project_id` + `domain`
5. **按时间范围查询**: `created_time`

#### 6.1.2 复合索引设计
```sql
-- 项目表复合索引
```
